# 发票扫描软件设计文档

**日期：** 2026-05-17
**项目：** InvoiceScan
**状态：** 已确认

---

## 1. 需求理解

构建一个 Windows 桌面应用，支持批量导入 PDF 和 PNG 格式的中国增值税发票文件，通过本地 PaddleOCR 提取结构化发票字段，用户可在界面中预览和手动修正识别结果，最终导出为 Excel/CSV 文件。

**核心约束：**
- 完全本地运行，数据不出本机
- 支持批量处理（文件夹导入 + 拖拽）
- 使用 PaddleOCR（中文发票识别准确率最优）
- UI 使用 PyQt6

---

## 2. 技术栈

| 组件 | 技术选型 |
|------|---------|
| UI 框架 | PyQt6 |
| OCR 引擎 | PaddleOCR（paddlepaddle + paddleocr） |
| PDF 转图片 | pdf2image + poppler（Windows 需将 poppler/bin 打包进 dist） |
| Excel 导出 | openpyxl |
| 测试框架 | pytest |
| 打包 | PyInstaller |
| Python 版本 | Python 3.9+ |

## 4. 数据流

```
用户选择/拖入文件（PDF 或 PNG）
    → 加入文件队列（主窗口列表显示，状态：待处理）
    → 点击"开始识别"
    → 后台线程逐个处理：
        PDF → pdf2image 转为图片
        图片 → PaddleOCR 提取文本
        文本 → invoice_parser 解析为结构化字段
        → Invoice 对象存入内存队列
    → 主窗口实时更新识别状态（待处理/识别中/完成/需复核/失败）
    → 识别完成后可在右侧面板预览、手动修正字段
    → 点击"导出 Excel" → openpyxl 写入文件，弹出保存对话框
```

## 5. 发票字段模型（Invoice）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `invoice_code` | str | 发票代码 |
| `invoice_number` | str | 发票号码 |
| `invoice_date` | str | 开票日期（YYYY-MM-DD） |
| `buyer_name` | str | 购买方名称 |
| `buyer_tax_id` | str | 购买方税号 |
| `seller_name` | str | 销售方名称 |
| `seller_tax_id` | str | 销售方税号 |
| `items` | list[InvoiceItem] | 货物/服务明细列表 |
| `subtotal` | Decimal | 合计金额（不含税） |
| `tax_rate` | str | 税率（如"13%"） |
| `tax_amount` | Decimal | 税额 |
| `total_amount` | Decimal | 价税合计 |
| `issuer` | str | 开票人 |
| `invoice_type` | str | 发票类型（增值税专票/普票） |
| `source_file` | str | 来源文件名 |
| `status` | str | 识别状态（success/review/failed） |
| `error_message` | str | 失败或需复核时的说明 |

**InvoiceItem 子结构：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `name` | str | 货物/服务名称 |
| `quantity` | str | 数量 |
| `unit_price` | Decimal | 单价 |
| `amount` | Decimal | 金额 |

## 6. UI 界面设计

### 主窗口布局（左右分栏）

```
┌─────────────────────────────────────────────────────────┐
│  [选择文件]  [选择文件夹]  [清空列表]        [开始识别]  │  ← 工具栏
├───────────────────────┬─────────────────────────────────┤
│  文件列表（左侧）      │  字段预览/编辑（右侧）           │
│                       │                                  │
│  □ invoice_001.pdf ✓  │  发票代码: 0440xxxxxxxxxx        │
│  □ invoice_002.png ✓  │  发票号码: 12345678              │
│  □ invoice_003.pdf ⚠  │  开票日期: 2024-03-15            │
│  □ invoice_004.png … │  购买方:   XX科技有限公司         │
│                       │  ...                             │
│                       │  [上一张]  [下一张]              │
├───────────────────────┴─────────────────────────────────┤
│  已完成: 3/4   失败: 0   需复核: 1      [导出 Excel]     │  ← 状态栏
└─────────────────────────────────────────────────────────┘
```

### 交互细节
- 支持**拖拽文件/文件夹**到主窗口直接加入队列
- 文件列表图标：`✓` 成功、`⚠` 需复核、`✗` 失败、`…` 处理中
- 右侧字段支持**手动编辑修正**，修改后标记为"已复核"
- 批量处理时弹出**进度条弹窗**，显示当前文件名和整体进度
- 导出时弹出对话框，选择"汇总模式"（每张发票一行）或"明细展开模式"（每条明细单独一行）

## 7. 错误处理策略

| 异常情况 | 处理方式 |
|----------|---------|
| PDF 加密或损坏 | 标记失败，显示错误原因，跳过继续处理 |
| 图片分辨率过低（< 72 DPI） | 自动放大后重试，仍失败则标记需复核 |
| OCR 置信度低（< 80%） | 标记需复核，提示用户手动确认 |
| 多页 PDF | 逐页识别，取包含关键字段的页面 |
| 关键字段缺失 | 对应字段留空并高亮提示 |
| 金额校验失败（合计 ≠ 不含税 + 税额） | 标记需复核 |
| 重复发票（同一发票代码 + 号码） | 弹窗提示用户确认是否重复 |
| 导出时 Excel 文件被占用 | 提示用户关闭后重试，不丢失数据 |

## 8. 测试策略

### 单元测试（pytest）
- `test_invoice_parser.py` — 用真实 OCR 文本样本测试字段提取逻辑
- `test_exporter.py` — 测试汇总模式和明细展开模式的 Excel 输出格式
- 金额校验逻辑、重复发票检测的边界用例

### 集成测试
- 准备 5-10 张真实增值税专票/普票样本（PDF + PNG），验证端到端识别准确率
- 批量处理 50 张发票的性能和稳定性测试

### 手动验收
- 拖拽文件夹导入 → 识别 → 手动修正字段 → 导出 Excel 全流程
- 加密 PDF、低分辨率图片等异常文件的处理表现
